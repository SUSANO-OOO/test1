# PAOPAO V29 REVIEW

更新日: 2026-07-11

## 改善目的

V29は、情報量を減らすのではなく、第三者が数秒で「何ができる人なのか」を理解できる構造へ改善する版です。既存のデザイン基盤を維持しながら、文字階層、ナビゲーション、専門領域、CURRENT、PROCESS、JOURNAL、CONTACTを再設計しました。

## 反映内容

- PC版ヘッダーナビの文字・クリック領域・選択状態を拡大
- HOMEの中心メッセージを「複雑な業務を、誰でも動ける仕組みに変える」へ変更
- 得意領域を、業務整理・ナレッジ化・問い合わせ運営の3領域として明示
- 経歴説明より前に「何ができるか」を表示
- カテゴリーカードをテンプレート的な均等カードから編集的なレイアウトへ変更
- PROCESSのPDCAを、独立した装飾ではなく仕事中に繰り返す改善循環として表現
- CURRENTから未実施のコミュニティ準備を削除
- CURRENTを以下の事実に限定
  - 独立に向けたオンラインスクールでの学習と実践
  - 男性向け減量コンテンツの設計
  - AIを制作支援として活用したポートフォリオサイトの制作
  - AIを活用した食事・ダイエット管理ツールの試作と改善
- JOURNALへ検索、並び替え、記事詳細、前後移動、共有リンクを追加
- CONTACTとInstagram導線の文字サイズ・クリック領域を拡大
- 背景の汎用的な光や星を弱め、情報構造を補助する表現を優先
- プロフィール画像をサイト内へ直接埋め込み、外部ファイル依存による画像切れを防止

## PDCAの扱い

Plan（計画）、Do（実行）、Check（評価・確認）、Act（改善）の循環として扱います。V29では「PDCAを使っています」と見せるための4分割カードにはせず、目的確認、初稿、確認、修正、更新という実務フローの中で必要な単位に分けて繰り返す考え方として記述します。

## QA

ローカルChromiumで以下を確認しました。

- 1440 × 900でのHOME表示とナビゲーション可読性
- プロフィール画像の読み込み
- CURRENTに「コミュニティ」の記載がないこと
- JOURNAL記事のクリックと詳細表示
- JOURNALの検索・並び替え・前後移動
- JavaScript構文エラーなし

## 未確認

- 実際のPATを使った本番JOURNAL書き込み
- 実機iPhone Safariでの最終操作
- 外部レンダラー上の表示は利用端末のスクリーンショットを最終基準とする
